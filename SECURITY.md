## Reporting a Vulnerability

If you discover a security vulnerability within this project, please send an email to Xuepoo. All security vulnerabilities will be promptly addressed.

### What to include in your report

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- Initial response: Within 48 hours
- Vulnerability assessment: Within 1 week
- Fix development: Within 2 weeks (depending on complexity)
- Public disclosure: After fix is released

## Security Best Practices

### For Users

1. Always download packages from official sources
2. Verify checksums when available
3. Keep your packages updated
4. Report any suspicious behavior

### For Contributors

1. Follow secure coding practices
2. Validate all inputs
3. Use parameterized queries
4. Keep dependencies updated
5. Review code for security issues

## Disclosure Policy

- Security vulnerabilities will be disclosed after a fix is available
- Credit will be given to reporters (unless anonymity is requested)
- CVE numbers will be requested for significant vulnerabilities
